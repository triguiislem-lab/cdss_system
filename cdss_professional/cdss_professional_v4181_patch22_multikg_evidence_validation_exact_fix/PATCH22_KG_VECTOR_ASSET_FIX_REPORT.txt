Patch22 MultiKG KG + Vector Asset Fix
=====================================

Context
-------
The uploaded builder/review notebooks show that the Hetionet/PrimeKG backup KG
and the medical vector fallback have concrete Kaggle packaging constraints that
must be handled by the project runtime, not only by endpoint probes.

Code changes
------------
1. Kuzu asset resolver now supports Kaggle ZIP packaging:
   - detects hetionet_primekg_kuzu_full.zip under the dataset root;
   - extracts to CDSS_KUZU_EXTRACT_DIR, defaulting to /kaggle/working/cdss_kuzu_backups;
   - resolves the real db.kuzu directory after extraction;
   - readiness reports the resolved extracted DB path.

2. Hetionet/PrimeKG Kuzu query compatibility:
   - supports RELATED relations with r.rel_type and r.source;
   - keeps fallback variants for older r.relation schemas;
   - supports node name, label, id and node_id fallbacks;
   - prevents the previous situation where readiness was green but backup_only returned zero facts.

3. Vector fallback compatibility:
   - VECTOR_FALLBACK_FAISS_STATS_PATH added;
   - vector_store_stats.json is used as source of truth for the FAISS embedding model;
   - avoids E5/BGE vs S-PubMedBert dimension mismatch on the backup FAISS index;
   - .env.kaggle and runtime envs point to vector_store_stats.json.

4. Notebook update:
   - stages/extracts the KG backup zip before endpoint probes;
   - records KG zip, resolved DB and vector stats paths in the source coverage report;
   - fixes backup ablation overlays so KG_SOURCE_MODE is primary_plus_backups / primary_plus_all_backups when backups are enabled.
